# 桩机双天线与桩心几何说明

本文档记录桩机双 RTK 天线与桩心 P 的几何关系。当前先按临时假设记录，后续拿到真实安装尺寸后再精确计算。

## 1. 当前确认的信息

桩机上有两根 RTK 天线：

```text
A = 左侧天线
B = 右侧天线
P = 桩心 / 钻杆中心 / 实际桩位
```

几何关系：

```text
        P
        |
        |
A-------M-------B
```

其中：

```text
M = AB 的中点
P 在 AB 的中垂线上
A/B 两根天线对称安装
```

因此当前不是用任意一根天线坐标作为桩心坐标，而是应该：

```text
A 天线坐标 + B 天线坐标 + 桩机固定几何尺寸 -> 计算 P 桩心坐标
```

## 2. 当前临时参数

先按临时值记录：

```text
AB = 3.000m
```

`AB` 表示两根天线相位中心之间的实际距离，不是外壳距离。

还需要后续确认：

```text
MP = AB 中点 M 到桩心 P 的实际距离
P 在 A -> B 方向的左侧还是右侧
```

如果现场把 A 定义为左天线、B 定义为右天线，则“左侧/右侧”需要结合坐标方向或设备前进方向统一定义，不能只凭页面视觉判断。

## 3. 计算思路

已知两根天线的成都项目坐标：

```text
A = (Ax, Ay)
B = (Bx, By)
```

先计算 AB 向量：

```text
dx = Bx - Ax
dy = By - Ay
L = sqrt(dx^2 + dy^2)
```

`L` 是实时测得的两天线间距。理论上它应接近真实 `AB`。

单位方向向量：

```text
ux = dx / L
uy = dy / L
```

中点：

```text
Mx = (Ax + Bx) / 2
My = (Ay + By) / 2
```

AB 的两个垂直方向：

```text
左侧法向  n1 = (-uy,  ux)
右侧法向  n2 = ( uy, -ux)
```

如果 P 在 A -> B 的左侧：

```text
Px = Mx - uy * MP
Py = My + ux * MP
```

如果 P 在 A -> B 的右侧：

```text
Px = Mx + uy * MP
Py = My - ux * MP
```

## 4. 为什么不能用比例

坐标系统和桩机尺寸都使用米：

```text
成都项目 X/Y 单位 = m
国家 2000 N/E 单位 = m
桩机安装尺寸也应使用真实 m
```

因此这里不应该用模拟比例，也不应该用屏幕比例。应该使用桩机现场真实测量长度：

```text
AB = 两天线相位中心距离
MP = 两天线中点到桩心距离
```

Canvas 只是显示层，不能参与物理几何比例换算。

## 5. 后续需要补充的数据

后续精确落算法前，需要确认：

```text
1. AB 真实长度，单位 m
2. MP 真实长度，单位 m
3. A/B 的固定命名：哪根是 A，哪根是 B
4. P 在 A -> B 的左侧还是右侧
5. 天线坐标是否是天线相位中心坐标
6. 双天线报文输出的是 A/B 各自坐标，还是一个主天线坐标 + 航向角
```

## 6. 推荐实现方式

后续建议新增一个纯函数：

```ts
type Point = {
  x: number;
  y: number;
};

function calcPileCenterFromAntennas(
  antennaA: Point,
  antennaB: Point,
  offsetFromMidpoint: number,
  side: "left" | "right"
): Point
```

这个函数只处理桩机几何，不处理坐标系转换。

完整链路应保持分层：

```text
RTK / 经纬度
-> 国家 2000 N/E
-> 成都项目 X/Y
-> 双天线 A/B 几何换算
-> 桩心 P 的成都项目 X/Y
-> Canvas 绘制
```

这样后续如果坐标系参数、桩机尺寸、Canvas 视图任意一层变化，都可以独立调整。
